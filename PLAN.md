## Phase 2: 디자인 시스템 구축 (1일)

### Task 2-1: HTML 구조 작성
- **파일**: `index.html`
- **내용**:
  - DOCTYPE, meta 태그 (반응형, charset)
  - Semantic HTML 구조
  - 각 섹션 (header, profile, contact, footer)
  - mailto 및 tel 링크
- **체크리스트**:
  - [ ] 기본 HTML 틀 작성
  - [ ] 프로필 섹션 마크업
  - [ ] 연락처 버튼 마크업
  - [ ] 유효한 HTML5 (W3C 검증)

### Task 2-2: 네오부루탈리즘 CSS 스타일 작성
- **파일**: `styles.css`
- **내용**:
  - 리셋 스타일 (Normalize)
  - 컬러 변수 정의
  - 타이포그래피 설정
  - 레이아웃 (Flexbox/Grid)
  - 네오부루탈리즘 요소들:
    - 굵은 테두리 (border: 3-4px solid #000)
    - 큰 여백 (padding, margin)
    - 고명도 대비
    - 기하학적 형태
  - 반응형 미디어 쿼리
  - 호버/활성 상태

- **체크리스트**:
  - [ ] 전체 레이아웃 구성
  - [ ] 컬러 시스템 적용
  - [ ] 타이포그래피 설정
  - [ ] 반응형 디자인 (Mobile First)
  - [ ] 버튼 스타일 (호버 효과 포함)
  - [ ] 모든 요소에 굵은 테두리 적용

## Phase 3: 기능 구현 (1일)

### Task 3-1: 정적 콘텐츠 조성
- **파일**: `index.html` 수정
- **내용**:
  - 프로필 정보 입력
  - 연락처 정보 입력
  - 유튜브 링크 추가
  - 이미지 경로 설정 (프로필 이미지)

- **체크리스트**:
  - [ ] 모든 정보 정확하게 입력
  - [ ] 링크 테스트 (mailto, tel, YouTube)
  - [ ] 이미지 최적화

### Task 3-2: JavaScript 인터랙션 (선택)
- **파일**: `script.js`
- **기능**:
  - 버튼 클릭 시 피드백 (옵션)
  - 복사 기능 (이메일, 전화)
  - 스무드 스크롤 (길이가 길 경우)

- **체크리스트**:
  - [ ] 필요시만 구현
  - [ ] 테스트 완료

### Task 3-3: QR 코드 생성 (선택)
- **라이브러리**: `qrcode.js`
- **내용**:
  - 디지털 명함 URL로 변환되는 QR 코드
  - 다운로드 기능

- **체크리스트**:
  - [ ] QR 코드 라이브러리 연결
  - [ ] QR 코드 이미지 생성
  - [ ] 스타일 적용

## 주의사항

### 디자인
- 네오부루탈리즘: 미니멀하면서도 대담해야 함
- 검은색과 흰색의 고대비 유지
- 불필요한 그림자나 둥근 모서리 피하기

### 기술
- 반응형 우선 (Mobile First)
- 단순하고 빠른 로딩
- 크로스 브라우저 호환성 필수

### 콘텐츠
- 모든 링크 정확성 확인
- 프로필 이미지 고해상도 준비
- 오타 없음 확인
